# Prep Plan — Aurora (Engineering Manager, Developer Platform)

## Constraints
- Days until interview: 10
- Rounds: recruiter screen (30m) → hiring-manager screen (45m: leadership background + management philosophy) → onsite (virtual): people-management case · cross-functional/conflict scenario · technical-direction/design discussion · values interview.
- **This loop is leadership-weighted, NOT a coding gauntlet.** Do not over-invest in algorithm/coding drills. Front-load behavioral stories, management philosophy, and the people/conflict cases.

## Priority queue (do in this order)
| # | Action | Skill to run | Maps to | Must-do? |
|---|---|---|---|---|
| 1 | Build the 5–7 STAR stories (leadership, disagreeing-up, conflict/coaching, failure-with-recovery, delivering-through-a-team) | build-stories | HM screen + onsite case/conflict/values | ✅ |
| 2 | Write a crisp management philosophy + the honest gap framing (influence → positional authority) | build-stories / cheatsheets | HM screen | ✅ |
| 3 | Close the performance-management gap: learn a feedback/review/PIP framework; rehearse a people-management case | build-curriculum | onsite people-management case | ✅ |

## Day-by-day cadence
- **Day 1:** Build the 5–7 STAR stories from the intake answers (the single biggest must-do — every round leans on these).
- **Day 2:** Draft + tighten the management philosophy and the one-line honest gap framing; map each story to an archetype.
- **Day 3:** research-prep — Aurora company brief, JD reverse-map, "why this company / why platform" angle.
- **Day 4:** Curriculum — performance management: a fair-feedback/review/calibration framework; write how you'd handle a low performer and a high performer.
- **Day 5:** Curriculum — hiring: how you'd run and calibrate a hiring loop as the hiring manager; build a debrief/close approach.
- **Day 6:** Rehearse the people-management case aloud; mock the cross-functional/conflict scenario.
- **Day 7:** Mock HM screen (leadership background + philosophy), graded by coach; fix the weakest answers.
- **Day 8:** Mock onsite people-management case + values; tighten stories to 90s each.
- **Day 9:** Light review — drill weak spots, re-rehearse the 2 headline stories, finalize smart questions per interviewer.
- **Day 10 / Morning-of:** Glance sheet — 90s pitch, 2 Aurora facts, 3 stories, why-Aurora, 3 questions. Rest.

## "Done" looks like
- Deliver all 5–7 stories cold, each in ≤90s, each landing on a concrete decision + a real number (18→6 min, ~40% incidents, 2 promoted to senior).
- State a coherent management philosophy in ≤60s and back each principle with a real episode.
- Name the formal-management gap in one honest sentence and immediately pivot to how you'll close it — no defensiveness.
- Walk through a people-management case (low performer, conflict, prioritization trade-off) using a named framework, not improvisation.
- Articulate how you'd run a hiring loop as hiring manager despite not having done it formally.
- Say "why Aurora / why this platform team" in their language, mapping your dev-platform + reliability wins to their charter.
