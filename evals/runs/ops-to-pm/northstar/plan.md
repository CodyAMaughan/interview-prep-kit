# Prep Plan — Northstar (Associate Product Manager)

## Constraints
- Days until interview: 10
- Rounds: recruiter screen (30m) → hiring-manager screen (45m: background + product-sense) → onsite (virtual): product-sense case · analytical/metrics case · execution case · behavioral

## Priority queue (do in this order)
| # | Action | Skill to run | Maps to | Must-do? |
|---|---|---|---|---|
| 1 | Build 5–7 STAR stories from real ops work + nail the honest career-switch framing | build-stories | recruiter + HM screen + behavioral onsite | ✅ |
| 2 | Company brief + HM/interviewer playbooks + JD reverse-mapping | research-prep | all rounds (esp. recruiter + HM) | ✅ |
| 3 | Close PM-craft gaps: product-sense framework, PRD structure, success-metrics tree, prioritization | build-curriculum | HM product-sense + all 3 onsite cases | ✅ |

| 6 | Timed mock: analytical/metrics case + light SQL drill | mock-interview / drill | onsite analytical case | ⬜ high-ROI |
| 7 | Execution-case prep (discovery → build → launch walkthrough) | build-curriculum | onsite execution case | ⬜ |

## Day-by-day cadence
- **Day 1:** build-stories — draft all 5–7 STAR stories (onboarding ambiguity, phased-rollout pushback, warehouse user-pain→prioritization, vendor consolidation, a failure/learning). Lock the 90-sec honest-switch pitch.
- **Day 2:** research-prep — Northstar brief (healthcare-workflow software), why-this-company angle (her healthcare-ops domain), recruiter + HM playbooks. Draft 3 smart questions.
- **Day 3:** build-curriculum — product-sense framework (user → problem → solution → metrics) + PRD skeleton. Apply to a Northstar-shaped problem.
- **Day 4:** build-curriculum — success-metrics / metrics-tree track + prioritization frameworks (RICE/impact-effort). Map her real ranked-issue-list instinct onto a named framework.
- **Day 5:** Rehearse all stories aloud to 90s; coach-grade the 1–2 ⭐ stories. Tighten the gap-acknowledgment language.
- **Day 6:** mock-interview — HM-style product-sense discussion, coach-graded. Fix the top 2 weaknesses.
- **Day 7:** build-curriculum / drill — analytical/metrics case practice + SQL beyond simple SELECTs (GROUP BY, joins).
- **Day 8:** mock-interview — analytical/metrics case + execution case, coach-graded.
- **Day 9:** Light review — re-run weak stories, re-read research brief, rehearse why-Northstar in her own words.
- **Day 10 / Morning-of:** cheatsheets glance sheet — pitch, 2 facts about Northstar, 3 questions, top 3 stories, the one-line honest framing.

## "Done" looks like
- She can defend every strong line cold with a real number: 14 clinics, 11→7 weeks, ~$220k/yr, ~1/3 fewer scheduling errors, 5 warehouse sites.
- She delivers the honest career-switch framing in one breath without apologizing, then pivots to transferable evidence.
- Each of the 5–7 stories lands in ≤90s on a concrete decision + a real number (or an honestly-flagged "needs a number").
- She can run a basic product-sense pass (user → problem → prioritized solution → success metric) on a healthcare-workflow problem unprompted.
- She can sketch a PRD skeleton and name a prioritization framework, mapping each to something she has actually done operationally.
- She can state why Northstar specifically — leaning on her healthcare-ops domain as the bridge — in Northstar's own language.
- She has 3 smart, role-specific questions ready and knows 2 concrete facts about Northstar.
