# Job description — Backend Software Engineer

> Modeled on real public mid-level backend postings. Fictional company "Foundry".
> Pairs with persona `no-numbers`.

**Foundry — Backend Software Engineer**
Remote (US) · Engineering

## About the role
Foundry builds tools for small businesses. We're hiring a backend-leaning full-stack engineer to build
customer-facing features and the services behind them, working closely with product and design.

## What you'll do
- Build and ship features across the stack, with an emphasis on backend services and APIs.
- Own the quality of what you ship: write tests, review code, and improve reliability.
- Collaborate with product and design from idea through launch.
- Participate in on-call and help improve performance and developer experience.

## Required
- 3+ years building web applications in production.
- Proficient with a modern backend stack (Node/TypeScript, Python, Go, or similar) and SQL.
- A track record of shipping features and owning their quality (testing, code review).
- Evidence of measurable impact on the products or systems you've worked on.

## Nice to have
- Front-end experience (React) for full-stack work.
- Experience improving performance or reliability with before/after results.
- AWS or another cloud in production.

## Interview process
1. Recruiter screen (30 min).
2. Technical screen (60 min): a coding problem + a discussion of past work.
3. Onsite (virtual): coding, system design, and a behavioral conversation.
