# Job description — Senior Platform Engineer, Data Infrastructure (FAKE example)

> Fictional company "Lumen Robotics". Invented for illustration only.

**Lumen Robotics — Senior Platform Engineer, Data Infrastructure**
Remote (US) · Platform team

## About the role
Lumen's robots generate a firehose of telemetry. Our Data Infrastructure team builds the streaming
platform that ingests, processes, and serves that data to fleet-ops, ML, and customer-facing products.
We're hiring a senior engineer to own core pieces of that platform and set technical direction for how
teams build on it.

## What you'll do
- Design and operate high-throughput streaming pipelines (millions of events/min) with strong
  reliability and observability guarantees.
- Build self-serve platform primitives so product teams can ship data products without re-inventing
  ingestion, schema management, and backfills.
- Set technical direction beyond your own work — mentor, review designs, and raise the bar.
- Run services on **Kubernetes**; manage infrastructure as code.

## Required
- 5+ years building backend or data systems in production.
- Deep experience with a streaming system (Kafka, Pulsar, Kinesis, or similar) and stateful processing.
- Strong fundamentals in **distributed systems** — partitioning, replication, consistency trade-offs.
- Fluent in **SQL** and at least one of Python / Go / Java.
- Comfortable owning reliability: alerting, on-call, runbooks, incident response.

## Nice to have
- **Kubernetes** in production; infra-as-code (Terraform).
- Exposure to ML serving / feature pipelines.
- Experience designing platform/self-serve abstractions used by other teams.

## Interview process
1. Recruiter screen (30 min, non-technical).
2. Technical screen (60 min): coding + a data/streaming design discussion.
3. Onsite (virtual): system design, a deep-dive on past work, and a values/leadership conversation.
